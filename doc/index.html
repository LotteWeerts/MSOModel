Core documentation¶

The core module contains the MSOModel class that runs the simulation.

class MSOModelModule.core.MSOModel(timesteps, batchsize, vt, input_neurons, MSO_neurons, dt, T, storeData=True, ITDoptions=[0.0], memorySavingMode=2, parallel=1, path=False, delays=False)[source]¶

This class contains the model of the MSO and methods to run simulations

The MSOModel is the main class that runs the simulation. See README.txt for instructions how to run it.

Parameters:
  • timesteps – The number of timesteps (in milliseconds) the simulation should run
  • dt – Stepsize of simulation (in ms)
  • vt – Threshold for voltage (theta v)
  • batchsize – The size of each batch (in milliseconds). For varying ITDs the batchsize determines how often the ITDs are changed.
  • input_neurons – Number of input neurons (layer 1)
  • MSO_neurons – Number of MSO neurons (layer 2)
  • T – The period of the input sound (i.e. determines the frequency)
  • storeData – Boolean that indicates whether or not simulation results should be stored
  • ITDoptions – An array of ITDs the simulation can choose from (set to e.g. [0.0] for constant ITD of 0.0)
  • path – Used for reloading model in a certain with MSOModel.loadLearnedModel(), don’t use yourself
  • delays – Used for reloading models with MSOModel.loadLearnedModel(), don’t use yourself
  • memorySavingMode – Determines the mode for memory saving. See note.

Note

Do not use memorySavingMode 0 or 1. These are only included for completeness (see appendix A). They are slower and take up more memory than memorySavingMode 2.

classmethod loadLearnedModel(folder, t)[source]¶

Class method that can be used to load a pre-existing model and test or run the model.

Parameters:
  • folder – Path to the folder where the model is stored
  • t – Time identifier of the weight matrix that should be loaded, i.e. if the filename is ‘19999_w.json.gz’ the t value is 19999

Note

The value for t is the time step, not the time in milliseconds. If you would like to reload a simulation after 500 ms for a simulation with dt = 0.005, you compute the correct weight matrix as follows:

500/0.005*1000 = 100000000

Test documentation¶

The test module contains helper functions that can be used to test the MSOModel and analyse its results.

MSOModelModule.test.ITDcorrelation(path, t, ITDoptions)[source]¶

ITD correlation runs an MSO model stored in a particular path for the given ITD options to test its responses for 10 ms. It stores the spikes of the test in a test folder that is created in the folder where the simulation is stored.

Parameters:
  • t – The time at which the simulation should be tested
  • ITDoptions – an array that contains all ITDs that should be tested
  • path – a string that refers to the path where the simulation is stored

Note

The parameter t is in timesteps, not in milliseconds. You compute the time you want to test the simulation on as t dt, where dt is the stepsize. This requires the existence of a file (t dt - 1)_w.json.gz in the data_per_batch section of the simulation folder
